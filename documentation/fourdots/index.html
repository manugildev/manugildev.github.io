Requirements & Installation

Requirements: 

1. Android Studio with Android SDK, Available for Windows and Mac (http://developer.android.com/sdk/index.html)

Import: 

For the purpouse of this tutorial I will do everything using Mac OS, but it is exactly the same if you are using Windows. Once you have extrated the app, open Androi Studio.

Once it is loaded, it will appear the "Welcome To Android Studio" window. Click on Import non-Android Studio Project, navigate with the explorer and go to the location of the folder you have just extracted, in my case I save it on my Desktop. Then click the root folder "Four Dots" and click "OK".

Now Android Studio it is going to build our project and then it will ask to change the SDK location, from my location to yours, and It will do it automatically, you just need to click Ok. This can take a long time if you don't have a fast computer. You have finished importing the project, your Android Studio should look something similiar to this:

Testing on Desktop

This game has been made using LibGDX which is a powerful Java Game Library, which makes things easier for the developers. Using Libgdx and Android Studio we can test our game on Desktop before exporting to and Android Application. We could almost try all functionalities except the ones that are related with Android, like Ads, Leaderboards and Achievements. If you want to try those features, Test on a real Android Device.

Testing on Desktop is really easy. Go to the tab near the Run Button (that green triangle at the top ), and click on Edit Configurations...

A window will now pop up, click on +, then on Application and you have to set the values exactly as I have it.

Main class: com.madtriangle.fourdots.desktop.DesktopLauncher (Click on the three dots and selec it)

Working Directory: pathToYourGameFolder/android/assets (Very important to set this)

Use classpath of module: desktop

Now close Apply to close the window and all the changes will be saved now. If you click on the Run button and you see the game running in desktop.

 

 

Testing on Android

Testing on android is as easy as on Desktop. Just open that tab again Edit Configurations, +, Android Application, and the settings should be something like this.

Then click on Apply and Run again but this time set Android in the Run Configuration selector, next to the Run button. Testing on Android takes more time than on desktop, remember that. A new window will appear, where you have to select the emulator or real device where you want to test it.

Just select the one you want and click Ok. Android Studio will build the debug apk for us and send it to your device.

Project Structure

There are 4 different main folders in our app, that we have to take care off, Android, Desktop, Core and PSD.

 

  • Desktop folder is used when you want to test on your desktop, you don't need to change anything there. 
  • Android folder is the whole android project that Android Studio will compile and build the App from. Some values need to be changed here. All the Android magic is here, Ads, Leaderboards, Achievements... 
  • Core folder is where the game runs, all the game logic is there, this folder is the most important of our project.
  • Psd folder is where I saved all the images that I use for the project, changing the graphics here will not affect the images of the game. Game assets are inside android > assets folder. 

 

Configuration File

Most of you, just want to change the colors and ids of Four Dots, that's why I created a Configuration file where you can change lots of variables of the game, like colors, velocities, ids and stuff like that. Just navigate to the Configuration.java file inside core > src > configuration > Configuration.java

More Settings can be modified from the file called Settings.java in the same folder as Configuration.java. The Settings.java file added in Four Dots makes the customization of the game easier than ever. Changing Game Name

If you want to change game name you have to change different files in the game. First of all go to your Configuration.java file and change the game name there. This will change the name on the Menu Screen of the game.

 

Then go to android folder, go to src and now you have to change the package name of the game that have to be unique so don't put the same as mine, google will not publish your app if the package name is the same as other one in the Play store. Basically you need to change where it says madtriangle which is my company name to yours, and fourdots to your game name. You need to have unchecked the option "Compact Empty Middle Packages", clicking on that gear button you will see this option. 

 

Right click on those folders, and click Refactor..., then change the name of the folder in the dialog that Android Studio opens. Once you have change the name of this two folders (you can also change the com folder name), go to android folder and AndroidManifest.xml file and check that the package name is also changed there to your new one. 

Sometimes Android Studio doesn't change it automatically, that's why you need to verify it.

Finally, go to android > res > values > strings.xml and change app_name there too. That's the name that appears below the icon in your Android Smartphone.

I would also recommend checking in android>build.gradle if all the package names there are the same as your current package name. That will solve most of the problems you can have on running the game on an Android Device.

Changing Images and Sounds

All the images and sounds used in the game are in a folder called android>assets, when you desing some graphics for the game throw all the stuff in that folder with the same name and you will change it in the game. Some of the images have a Photoshop file in the PSD folder.

If you are going to use textures replacing the dots, make sure you check the option USE_TEXTURES = true; in the Settings.java file and replace the dots in the file dots.png, they will be replaced for the color dots automatically.

 

ADS

Go to the Configuration.java file and change the values inside quotes for Banner and Interstitial ads. You can also change the frecuency of the ads to appear once you finish the game, in the game over screen.

LEADERBOARDS

Go to Configuration.java file and set your ids there for the different Leaderboards.

You will also need to set the app_id (you get it for the developer console) inside android > res > values > strings.xml.

IN-APP PURCHASES

In-app purchases can be also configurated in the Configuration.java file. You can set them to true or false, depending on your needs.

If you wanna activate them, set that variable to true, and you only need to copy your keys, obtained from the Google Play Developer Console.

  • Product_ID (Click on it to read more about it)
  • ENCODED PUBLIC KEY also called License Key

For the Product_ID, you must create an Unmanaged Porduct by clicking on In-App Purchases > Add new Product, you can see my configuration here.

I set the ID of my product to be "removeads", and that's the value you need to save and later you will add it to the Configuration File. Now we are going to get the  Public Key.

Once you have those 2 IDs, set them up in the Configuration.java file. For Four Dots, the product ID that I have used is "removeads" for removing the ads on the game when the user buys. If you set the IAP_ON to false, the button will be removed.
